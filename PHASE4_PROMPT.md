## What to Build

### Step 1: Data Models

Create `~/projects/petflix/Petflix/Core/Models/EpisodeScene.swift`:

```swift
import Foundation

/// Defines Ken Burns pan/zoom effect parameters for a scene
struct KenBurnsEffect: Sendable {
    enum Easing: Sendable {
        case linear, easeIn, easeOut, easeInOut
    }
    enum PanDirection: Sendable {
        case leftToRight, rightToLeft, topToBottom, bottomToTop, none
    }
    var startScale: CGFloat   // e.g. 1.0
    var endScale: CGFloat     // e.g. 1.15

/// Defines a text overlay that appears during a scene

    var text: String
    var fontName: String       // e.g. "Cinzel-Bold"
    var fontSize: CGFloat      // e.g. 48
    var color: String          // hex e.g. "#FF0080"
    var position: Position     // where on screen
    var fadeInDuration: Double  // seconds
    var fadeOutDuration: Double
    
    enum Position: Sendable {
        case center, bottomCenter, topCenter, bottomThird
    }
}

/// Transition between scenes
enum TransitionType: Sendable {
    case crossfade(duration: Double)  // duration in seconds
    case hardCut
}

/// One scene in an episode
struct EpisodeScene: Sendable {
    var imageName: String            // filename (e.g. "scene-01.png")
    var duration: TimeInterval       // seconds this scene displays
    var narrationText: String?       // TTS text (Phase 5, nil for now)
    var textOverlay: TextOverlay?    // optional text on screen
    var kenBurns: KenBurnsEffect?    // optional Ken Burns (nil for Phase 4-lite)
    var transitionToNext: TransitionType  // how to transition to next scene
}
```

### Step 2: macOS Test Script

Create `~/projects/petflix/test-video-assembly.swift` — a standalone macOS
command-line Swift script that:

1. **Loads the 8 scene PNGs** from `~/projects/petflix/test-outputs/episode-throne-1/`
   as NSImage → CGImage (this is macOS, use AppKit not UIKit)

2. **Creates an AVAssetWriter** configured for:
   - Output: `~/projects/petflix/test-outputs/episode-throne-1/episode-test.mp4`
   - Video codec: H.264 (AVVideoCodecType.h264)
   - Resolution: 1080x1920 (9:16 portrait)
   - Frame rate: 30 fps

3. **For each scene image (8 total):**
   a. Scale and center-crop the image to fill 1080x1920 (aspect fill)
   b. Convert to CVPixelBuffer (BGRA format)
   c. Write the same frame repeatedly for the scene duration at 30fps
      - Scene duration: 8 seconds = 240 frames per scene
   d. For crossfade transitions between scenes:
      - Over the transition duration, blend the outgoing scene's last
        frames with the incoming scene's first frames using alpha lerp
      - The transition frames REPLACE the end of scene N and start of
        scene N+1 (don't add extra time)

4. **Use these scene configurations** (from MICRODRAMA_E2E_TEST.md):

| Scene | Duration | Transition to Next |
|-------|----------|--------------------|
| 1     | 8s       | crossfade 0.8s     |
| 2     | 8s       | crossfade 0.8s     |
| 3     | 8s       | hard cut           |
| 4     | 8s       | crossfade 1.0s     |
| 5     | 8s       | crossfade 1.5s     |
| 6     | 8s       | hard cut           |
| 7     | 8s       | crossfade 0.8s     |
| 8     | 8s       | none (last scene)  |

5. **Finalize** the AVAssetWriter and print:
   - Output file path
   - Total video duration
   - Assembly time (wall clock)
   - File size

6. **Delete the output file first** if it already exists (so re-runs work)

## Implementation Notes

- This is a macOS script, not iOS. Use `import AppKit` for NSImage, 
  `import AVFoundation` for video writing, `import CoreVideo` for pixel buffers.
- Compile and run: `swift ~/projects/petflix/test-video-assembly.swift`
  - If swift compiler complains about frameworks, use:
    `swiftc -framework AVFoundation -framework CoreVideo -framework AppKit -framework CoreImage ~/projects/petflix/test-video-assembly.swift -o /tmp/test-video-assembly && /tmp/test-video-assembly`
- CVPixelBuffer creation: use `CVPixelBufferCreate` with `kCVPixelFormatType_32BGRA`,
  lock the base address, draw the CGImage into it via CGContext, unlock.
- For crossfade blending: create a CIContext, use CIFilter "CIBlendWithAlphaMask"
  or manually blend pixel values. The simpler approach: for each transition frame,
  compute alpha (0.0 → 1.0 over the transition duration), draw scene N at
  (1-alpha) opacity and scene N+1 at alpha opacity into a new CGContext,
  convert to CVPixelBuffer.
- AVAssetWriter timing: use `CMTimeMake(value: frameNumber, timescale: 30)`
  for each frame's presentation time.
- The pixel buffer pool from AVAssetWriterInputPixelBufferAdaptor is preferred
  over creating new CVPixelBuffers each frame.
- Handle the adaptor's `assetWriterInput.isReadyForMoreMediaData` — if false,
  wait briefly before retrying.
- Print progress every scene: "Assembling scene 3/8..."
- Total expected video: ~64 seconds (8 scenes × 8s, minus transition overlaps)
- Target: Assembly completes in under 15 seconds on M4 Pro.

## What NOT to do

- Do NOT use UIKit (this is macOS)
- Do NOT add audio/music (that's Phase 5)
- Do NOT add Ken Burns motion (that's Phase 4 full — we're doing 4-lite first)
- Do NOT add text overlays yet (that's Phase 4 full)
- Do NOT try to load images from an Xcode asset catalog
- Do NOT create an Xcode project — this is a standalone Swift script

## Success Criteria

After running the script:
1. `~/projects/petflix/test-outputs/episode-throne-1/episode-test.mp4` exists
2. Opening it in QuickTime shows all 8 scenes playing sequentially
3. Crossfade transitions are visible between scenes (smooth blend, not jarring)
4. Hard cuts at scenes 3→4 and 6→7 are instant (no blend)
5. Total duration is approximately 60-64 seconds
6. Video is 1080x1920 portrait orientation
7. Assembly completed in under 15 seconds
